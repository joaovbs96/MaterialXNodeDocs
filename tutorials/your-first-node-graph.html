Your first node graph¶

This walkthrough builds a simple red material from scratch in the Node Graph Editor — the same material as the minimal example in Getting started, but constructed by hand so you get a feel for the editor itself.

1. Open the Node Graph Editor¶

Open the Node Graph Editor with a blank document (use "New" if you already have something loaded).

2. Add a standard_surface node¶

Open the node palette and search for standard_surface. Drag it onto the canvas — this is the shader node that defines the material's surface response (base color, roughness, metalness, and so on).

3. Set the base color to red¶

Select the standard_surface node and open its inputs panel. Set:

  • base to 1.0
  • base_color to (0.8, 0.1, 0.1)

Edit the base_color input directly on the standard_surface node — click the color swatch and pick a red, or type the RGB values.

The live 3D preview updates immediately as you edit the input — no separate "render" step.

4. Add a surfacematerial node¶

Search the palette for surfacematerial and drag it onto the canvas. Connect the standard_surface node's output to the surfacematerial node's surfaceshader input by dragging from one port to the other.

Note

surfacematerial is what makes the shader assignable to geometry — without it, the graph defines a shader but not a renderable material. This mirrors the standard_surface + surfacematerial pair in the minimal .mtlx example.

5. Check the result¶

With surfacematerial connected, the preview panel renders your red material on the default preview mesh.

Want to see the underlying XML?

Use the Document view (XML view/export) in the graph editor's toolbar to see the exact .mtlx markup your graph produced — it should look very close to the hand-written example in Getting started.

6. Save your work¶

Export or save the document from the graph editor's toolbar to get a .mtlx file you can reopen later, load in the Material Viewer, or share.

Next steps¶

  • Browse the Node Library for other shader and pattern nodes to try (texture lookups, noise, math operations, ...).
  • Try nesting a sub-graph as its own nodegraph for reusable pieces.
  • Explore the implementation-target matrix on any node's documentation page to see which renderers support it.
